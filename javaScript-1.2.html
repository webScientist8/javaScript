一、变量提升

变量提升：在当前作用域中的JS代码自上而下执行之前,浏览器首先会把当前作用域中所有带var和function关键字的,进行提前的声明或者定义,这种机制就是变量提升(预解释)

  • 声明（declare）：告诉当前作用域有这样一个变量
  • 定义（defined）：给变量赋值

VAR和FUNCTION的区别：

  • var：只是提前的声明，不定义（默认值是undefind）
  • function：声明+定义

特：声明(var)+定义(赋值)=完整的创建

特：如果函数里的某个变量没有var，而全局下也没有，则相当于给全局增加了一个属性

 function fn() {

        n = 7;//->全局下也没有N这个变量,此时相当于给window增加了一个n的属性,属性值是7

    }

    fn();

    console.log(n);//->7

在全局作用域下声明的变量，相当于给全局对象window增加了一个属性（私有作用域不具备这个特点）

 var a=12;  /<=>window.a=12;

    console.log(a);  /->12  输出变量存储的值

    console.log(window.a);  /->12  输出win属性a的属性值  window.a

    b=13;  /->b不是变量，它是window的一个属性名（前提：在全局作用域下操作）  window.b=13；

    console.log(b);  /->13  window.b

    console.log(window.b);  /->  13  window.b
IF判断的变量提升：

不管条件是否成立，判断体中的内容也要参与变量提升（但是此时不管是var/function都只是提前生命不再定义了）；在代码执行过程中，如果条件不成立直接跳过判断继续执行，如果条件成立，进入到判断体中的第一件事情就是给函数赋值定义

不管条件是否成立都要进行预解释(IF和FOR在ES5的机制下是不存在作用域概念的,ES6的机制下是块级作用域)

     *  ->不管是带VAR还是带FUNCTION,判断体中出现的内容,在变量提升阶段都仅仅是声明,没有定义

     *
in判断：判断当前的属性是否属于这哥对象

‘xxx’ in window：in是用来判断当前属性是否隶属于这个对象的，属于返回true，反之false

function fn(){console.log(1);}

var fn=13;

window.fn=函数地址

window.fn=13;

不管是函数名还是变量名，其实都是一样的，都可以叫做变量名，如果名字一样了，也算是重复的

在变量提升的阶段，如果当前的这个名字已经被声明过了，没必要在重复声明，只需要给它赋值定义即可，如下：↓

console.log(fn);//->输出：function fn(){console.log(2);}

function fn(){console.log(1);};

function fn(){console.log(2);};

二、作用域

1、函数执行的步骤
  • 1，形成一个新的私有作用域,供函数体中的代码执行
  • 2，在私有作用域JS代码自上而下执行之前
    • 给形参赋值
    • 私有作用域下的变量提升
    • 私有作用域中的js代码自上而下执行
  • 3,作用域释放或者不释放
    ↑以上有个专业说法==>”函数执行的生命周期”

在私有作用域中
哪些是私有变量:

  • 1，形参变量
  • 2，提前声明的变量或者函数
    这两个都是“私有变量”，私有变量和外界是没关系的不冲突的

函数执行形成一个私有的作用域，隔离保护了里面的私有变量，使其和外界互不干扰（作用：避免了和全局变量的冲突，防止了全局变量的污染）我们把这种机制叫做——“闭包”

一个函数中出现的变量不一定都是“私有的变量”，如果遇到的变量不是私有的，则向当前作用域的上级作用域进行查找，看是否为上级作用域私有的，如果不是继续查找，一直找到window为止。。。

我们把这种查找机制叫做——“作用域链”

细节：

自执行函数(或者匿名函数)：函数创建和函数执行都完成了（自执行函数不参与变量提升）

私有作用域里使用变量:先用私有作用域里的变量，私有作用域里没有变量，才调用外面的同名变量，如果想直接调取，前面要加window.变量名
代码里的创建函数，不执行，跳过，因为变量提升里面已经把函数创建了，

在变量提升阶段，如果当前这个名字已经声明过了，则不再重新的声明，只需要重新的定义赋值即可 (变量名和函数名如果相同也算是重复)

var f=function(){return false;}//函数表达式；只声明不定义，符号代码自上而下执行的逻辑，项目中常用；

function g(){return true;}//g=函数本身

匿名函数：函数表达式(把函数当做一个值,赋值给一个变量或者某个元素的某个事件)

document.body.onclick=function(){};

document.body.onclick();

JS中的同步编程和异步编程
同步：
自上而下依次执行,上面的事情没有处理完,下面事情一直在等待
异步:
当上面的事情在等待执行的时候,我们不等,继续执行下面的事情，JS中的事件绑定属于异步编程

2、作用域【scope】

全局作用域：浏览器加载页面就形成的(window 浏览器加载页面的时候形成全局作用域)

私有作用域：函数执行形成的私有作用域

不管是全局的还是私有的作用域，都是给js代码提供运行环境的，我们把作用域称之为栈内存

栈内存：（作用域）用来执行代码的那个作用域环境

栈内存释放：全局作用域需要在关掉页面的时候自动销毁
一般情况下，函数执行完成，浏览器也会把当前的私有作用域自动释放销毁

不销毁的作用域：当栈内存中的某一样东西（一般都是对象或者函数的堆内存）被栈内存以外的变量（只能是其他作用域）给占用了，

所有的引用数据类型在定义的时候，都会开辟一个新的内存空间，来存储自己的值，我们把存储东西的内存空间称之为堆内存

  • 对象在自己的堆内存中存储的是“键值对”
  • 函数在自己的堆内存中存储的是“代码字符串”

堆内存：浏览器给引用数据类型开辟的那个空间（作用：存储东西的）
->对象存储的是键值对
->函数存储的是代码字符串
堆内存释放：

如果当前堆内存地址被其它的变量占用了（不管是当前作用域的，还是其他作用域的），堆内存是不能销毁的，如果作用域销毁了，里面的一切也都销毁了，变量跟堆内存自然也销毁了，

全局变量和私有变量
私有变量：

  • 函数的形参是私有作用域中的私有变量
  • 在私有作用域中声明过的变量或函数都是私有的

全局变量

  • 除了私有变量其它都是全局变量
  • 在全局作用域中声明的变量肯定是全局变量

函数执行形成一个私有的作用域（栈内存）
1，形参赋值
2，变量提升
形参或者是声明过的变量是私有的变量，否则不是私有的

私有的变量收到了保护，和外界的变量没有关系，不会收到任何的干扰，我们把函数执行的这种保护机制叫做“闭包”

在私有作用域中执行代码的时候，如果当前这个变量不是私有变量，则向上一级查找，也不是上一级私有的，则继续向上查找，一直找到window。。。，我们把这种向上一层层查找的机制叫做“作用域链”
